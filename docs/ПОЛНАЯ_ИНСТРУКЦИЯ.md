## 🗄️ БАЗА ДАННЫХ — Supabase + PostgreSQL

### Шаг 1: Создать аккаунт Supabase
1. Зайдите на https://supabase.com
2. Нажмите **Start your project**
3. Войдите через GitHub
4. Нажмите **New project**
5. Введите:
   - Name: `dayana-qyz-uzatu`
   - Database Password: (придумайте надёжный пароль)
   - Region: `Frankfurt (eu-central-1)` — ближайший к Казахстану

### Шаг 2: Создать таблицы

В Supabase → **SQL Editor** выполните следующий SQL:

```sql
-- Таблица гостей RSVP
CREATE TABLE rsvp_responses (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  created_at TIMESTAMPTZ DEFAULT NOW(),
  
  -- Основная информация
  full_name TEXT NOT NULL,
  attending BOOLEAN NOT NULL,
  language TEXT DEFAULT 'ru',
  
  -- Сопровождающие
  guest_count INTEGER DEFAULT 0,
  companions TEXT[] DEFAULT '{}',
  
  -- Дополнительно
  comment TEXT,
  
  -- Уведомления
  notification_sent BOOLEAN DEFAULT FALSE,
  reminder_sent BOOLEAN DEFAULT FALSE
);

-- Индексы для быстрого поиска
CREATE INDEX idx_rsvp_attending ON rsvp_responses(attending);
CREATE INDEX idx_rsvp_created ON rsvp_responses(created_at DESC);

-- Просмотр: статистика
CREATE VIEW rsvp_stats AS
SELECT 
  COUNT(*) FILTER (WHERE attending = true) as confirmed_yes,
  COUNT(*) FILTER (WHERE attending = false) as confirmed_no,
  COALESCE(SUM(guest_count) FILTER (WHERE attending = true), 0) as total_companions,
  COUNT(*) as total_responses
FROM rsvp_responses;

-- RLS политики (Row Level Security)
ALTER TABLE rsvp_responses ENABLE ROW LEVEL SECURITY;

-- Разрешить вставку всем (анонимным пользователям)
CREATE POLICY "Allow public insert" ON rsvp_responses
  FOR INSERT TO anon
  WITH CHECK (true);

-- Просмотр только через service_role (для Telegram бота)
CREATE POLICY "Allow service read" ON rsvp_responses
  FOR SELECT TO service_role
  USING (true);
```

### Шаг 3: Получить ключи API

В Supabase → **Settings → API**:
- `Project URL` → `NEXT_PUBLIC_SUPABASE_URL`
- `anon / public key` → `NEXT_PUBLIC_SUPABASE_ANON_KEY`  
- `service_role key` → `SUPABASE_SERVICE_ROLE_KEY` (секретный!)

## 🤖 УВЕДОМЛЕНИЯ — Telegram Bot

### Почему Telegram, а не WhatsApp?

| Критерий | Telegram | WhatsApp |
|----------|----------|----------|
| API для ботов | ✅ Бесплатно | ❌ Платно ($50+/мес) |
| Сложность | ✅ Просто | ❌ Сложно |
| Скорость | ✅ Мгновенно | ✅ Мгновенно |
| Надёжность | ✅ Высокая | ✅ Высокая |
| Вывод | ✅ **Telegram** | ❌ |

### Создание Telegram бота

1. Откройте Telegram, найдите **@BotFather**
2. Напишите `/newbot`
3. Введите имя: `Dayana Qyz Uzatu Bot`
4. Введите username: `dayana_qyzuzatu_bot` (или любое свободное)
5. Скопируйте **токен**: `7123456789:AAH...`

### Получить Chat ID для уведомлений

1. Создайте группу в Telegram (например "Даяна RSVP")
2. Добавьте бота в группу
3. Напишите любое сообщение в группе
4. Откройте: `https://api.telegram.org/bot<TOKEN>/getUpdates`
5. Найдите `"chat":{"id":-100...}` — это ваш Chat ID

5. Подождите 5-30 минут — DNS обновится

## 📲 НАСТРОЙКА TELEGRAM УВЕДОМЛЕНИЙ

### Формат уведомления при новом RSVP

```
🌸 Новый RSVP — Қыз Ұзату Даяны

👤 Гость: Иванова Мария Петровна
✅ Будет: ДА
👥 Сопровождающих: 2
   • Иванов Иван
   • Иванова Алина
💬 Комментарий: Поздравляем!
🗓 Время: 15 июня 2026, 14:32
```

### API Route для уведомлений

Файл `src/app/api/rsvp/route.ts`:

```typescript
import { createClient } from '@supabase/supabase-js'
import { NextRequest, NextResponse } from 'next/server'

const supabase = createClient(
  process.env.NEXT_PUBLIC_SUPABASE_URL!,
  process.env.SUPABASE_SERVICE_ROLE_KEY!
)

async function sendTelegramNotification(data: any) {
  const { full_name, attending, guest_count, companions, comment } = data
  
  const attendingEmoji = attending ? '✅' : '❌'
  const attendingText = attending ? 'ДА' : 'НЕТ'
  
  let companionsList = ''
  if (companions?.length > 0) {
    companionsList = '\n' + companions.map((c: string) => `   • ${c}`).join('\n')
  }
  
  const message = `
🌸 <b>Новый RSVP — Қыз Ұзату Даяны</b>

👤 <b>Гость:</b> ${full_name}
${attendingEmoji} <b>Будет:</b> ${attendingText}
👥 <b>Сопровождающих:</b> ${guest_count}${companionsList}
${comment ? `💬 <b>Комментарий:</b> ${comment}` : ''}
🗓 <b>Время:</b> ${new Date().toLocaleString('ru-RU', { timeZone: 'Asia/Almaty' })}
  `.trim()

  await fetch(
    `https://api.telegram.org/bot${process.env.TELEGRAM_BOT_TOKEN}/sendMessage`,
    {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({
        chat_id: process.env.TELEGRAM_CHAT_ID,
        text: message,
        parse_mode: 'HTML',
      }),
    }
  )
}

export async function POST(request: NextRequest) {
  try {
    const body = await request.json()
    const { full_name, attending, guest_count, companions, comment, language } = body

    // Сохранить в базу данных
    const { data, error } = await supabase
      .from('rsvp_responses')
      .insert({
        full_name,
        attending,
        guest_count: guest_count || 0,
        companions: companions || [],
        comment,
        language,
      })
      .select()
      .single()

    if (error) throw error

    // Отправить уведомление в Telegram
    await sendTelegramNotification(body)

    return NextResponse.json({ success: true, id: data.id })
  } catch (error) {
    console.error('RSVP error:', error)
    return NextResponse.json({ success: false, error: 'Server error' }, { status: 500 })
  }
}
```

```typescript
// Запускается 26 августа в 10:00
// Отправляет напоминание всем подтвердившим гостям
export async function GET() {
  // Получить всех гостей с attending = true

    .eq('reminder_sent', false)
```
